Setup your Okta Application

If you wish to run a debug login, you will need:

  • An Okta web application profile which is Open ID Connect enabled
  • The Client ID of that application
  • The Client Secret of that application
  • In the General Settings > Login > Sign-in redirect URIs section, you should add http://{host-where-your-running-this}/okta/callback

Replace the following values in the application.properties file:

  • sample.okta.app.clientId with your app Client ID
  • sample.okta.app.clientSecret with your app Client Secret

Restart the application and try the sign in flow again.
